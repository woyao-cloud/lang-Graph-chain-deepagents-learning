● HelloDeepAgents01.py 分析

  功能概述

  这是一个基于 deepagents 库和 LangChain 的 Deep Research Agent demo，使用
  Tavily 进行网络搜索。

  核心结构

  ┌────────────┬────────────────────────────────────────────────────────────┐
  │    组件    │                            说明                            │
  ├────────────┼────────────────────────────────────────────────────────────┤
  │ LLM        │ ChatOpenAI + DashScope 兼容接口 (DeepSeek-v3.2-exp / Qwen) │
  ├────────────┼────────────────────────────────────────────────────────────┤
  │ 搜索工具   │ TavilyClient (互联网搜索)                                  │
  ├────────────┼────────────────────────────────────────────────────────────┤
  │ Agent 框架 │ deepagents 的 create_deep_agent                            │
  └────────────┴────────────────────────────────────────────────────────────┘

  代码流程

  1. 初始化 TavilyClient → 封装 internet_search 工具
  2. 构造 system_prompt → 定义 researcher 角色
  3. 初始化 ChatOpenAI LLM (api_key + base_url)
  4. create_deep_agent(model, tools, system_prompt)
  5. agent.invoke({"messages": [...]}) → 执行推理

  潜在问题

  1. API Key 硬编码 — TAVILY_API_KEY 和 OPENAI_API_KEY
  直接写在代码中（应使用环境变量）
  2. 模型名特殊 — deepseek-v3.2-exp 是实验性模型，非稳定版本
  3. 缺少错误处理 — invoke 调用没有 try/except，API 失败会直接抛异常
  4. 单次调用 — 没有流式输出、增量展示或后续迭代逻辑

  可改进点

  - 添加 stream=True 支持流式输出
  - 增加 max_tokens 限制防止过量输出
  - 用 MessagesPlaceholder 支持多轮对话
  - 添加搜索结果的摘要/过滤逻辑

  需要我针对某个方面深入分析或提出具体改进方案吗？

     DeepAgents Code Agent 实现规划

     Context

     目标: 基于 deepagents 框架和
     LangChain/LangGraph，实现一个完整的多智能体代码生成系统（Code Agent），参考
      HelloDeepAgents01.py 的实现方式。

     需求来源:
     - requirments/SPEC.md — 8大功能模块（FR-WF/FR-PLAN/FR-AGENT/FR-EXEC/FR-PROG
     RESS/FR-HITL/FR-QA/FR-VCS）+ 非功能需求
     - docs/ARCHITECTURE.md — 详细架构设计（Multi-Agent
     DAG、Memory分层、Token预算、人机协同）
     - HelloDeepAgents01.py — deepagents 框架使用示例

     核心差距分析: 当前只有 demo 代码，缺少完整的工程实现。

     ---
     Phase 1: 核心基础设施

     1.1 CLI 框架搭建

     目标: 实现 deepagents CLI 命令行工具

     ┌────────────────────────┬────────────────────────────────────────────────┐
     │          命令          │                      功能                      │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents init        │ 初始化项目（生成 workflow.md, agent.md）       │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents run         │ 启动执行（支持 --phase, --parallel, --watch,   │
     │                        │ --resume）                                     │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents confirm     │ 确认规划                                       │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents status      │ 查看状态（--live 实时）                        │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents logs        │ 查看日志（--follow）                           │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents             │ 人工干预                                       │
     │ skip/rollback          │                                                │
     ├────────────────────────┼────────────────────────────────────────────────┤
     │ deepagents             │ 危险操作确认                                   │
     │ approve/reject         │                                                │
     └────────────────────────┴────────────────────────────────────────────────┘

     关键文件:
     - deepagents/cli.py — CLI 入口（使用 Click/Typer）
     - deepagents/commands/ — 各命令实现

     验收: deepagents --help 输出所有命令

     1.2 工作流解析模块 (FR-WF-001)

     目标: 解析 workflow.md → 构建 DAG

     关键文件:
     - deepagents/workflow/parser.py — Markdown/YAML 解析器
     - deepagents/workflow/dag.py — DAG 构建器（拓扑排序、并行节点识别）
     - deepagents/workflow/navigator.py — 阶段推进器

     验收:
     - 给定 workflow.md 能解析所有 Phases/Tasks
     - 给定 parallel: true 能识别可并行任务
     - DAG 拓扑排序正确

     1.3 配置文件生成

     目标: deepagents init 生成标准配置文件

     生成文件:
     - workflow.md — 默认模板（Phase 1: 架构设计，Phase 2: 开发，Phase 3: 测试）
     - agent.md — 默认角色定义（architect, backend-dev, frontend-dev,
     qa-engineer）

     ---
     Phase 2: Agent 核心

     2.1 Main Agent (Supervisor) 实现

     目标: 实现主 Agent（解析配置、生成规划、调度 Sub-Agent）

     关键文件:
     - deepagents/agents/supervisor.py — 主 Agent（基于 LangGraph）
     - deepagents/agents/planning.py — 规划生成模块
     - deepagents/prompts/supervisor.py — 主 Agent system prompt

     核心流程:
     workflow.md + agent.md → Supervisor Agent → PLANNING.md → 人工确认 →
     调度执行

     2.2 Sub-Agent 角色定义

     目标: 根据 agent.md 动态实例化 Sub-Agent

     关键文件:
     - deepagents/agents/roles/architect.py — 架构师 Agent
     - deepagents/agents/roles/backend_dev.py — 后端开发 Agent
     - deepagents/agents/roles/frontend_dev.py — 前端开发 Agent
     - deepagents/agents/roles/qa_engineer.py — QA Agent

     关键实现: 使用 create_deep_agent 工厂函数，根据 agent.md 动态绑定 tools 和
     prompts

     2.3 多 Agent 调度器 (FR-AGENT-001)

     目标: 基于 DAG 的并行/串行调度

     关键文件:
     - deepagents/scheduler/dispatcher.py — 任务分发器
     - deepagents/scheduler/executor.py — 并行/串行执行器
     - deepagents/scheduler/context_isolation.py — 上下文隔离

     核心逻辑:
     - 识别 DAG 中无依赖的叶子节点
     - 并发调度多个 Sub-Agent
     - 每个 Sub-Agent 运行在独立上下文

     ---
     Phase 3: 工具与记忆

     3.1 工具层 (FR-EXEC-001)

     目标: 实现 LangChain Tool 规范的工具集

     关键文件:
     - deepagents/tools/file_ops.py — 文件读写编辑工具
     - deepagents/tools/command.py — 命令执行工具（白名单）
     - deepagents/tools/git_ops.py — Git 操作工具
     - deepagents/tools/search.py — 代码搜索工具

     命令白名单:
     - 文件操作: read_file, write_file, edit_file, glob, grep
     - Git: git status, git add, git commit, git branch
     - 危险操作（需二次确认）: rm -rf, sudo, git push --force

     3.2 记忆管理系统 (FR-EXEC-001.3)

     目标: Working Memory + Short-term + Long-term 分层

     关键文件:
     - deepagents/memory/working.py — 工作记忆（单次对话）
     - deepagents/memory/short_term.py — 短期记忆（单日会话）
     - deepagents/memory/long_term.py — 长期记忆（跨会话）
     - deepagents/memory/retrieval.py — 语义检索

     Token 预算管理 (NF-PERF):
     - 全局预算: 200K tokens
     - Per-Agent 预算: Main 100K, Architect 80K, Backend-Dev 60K
     - 自动压缩阈值: 80%

     Phase 4: 质量与人机协同

     4.1 质量门禁 (FR-QA-001)

     目标: 自动化 Lint/Test/Schema 检查

     关键文件:
     - deepagents/quality/linter.py — Lint 检查
     - deepagents/quality/test_runner.py — 单元测试执行
     - deepagents/quality/schema_check.py — API Schema 校验
     - deepagents/quality/reporter.py — 质量报告生成

     质量门禁触发点:
     - 每阶段完成后
     - 失败时自动修订 PLANNING.md

     4.2 人机协同 (FR-HITL-001)

     目标: 关键节点人工干预

     关键文件:
     - deepagents/hitl/pause_manager.py — 暂停点管理器
     - deepagents/hitl/intervention.py — 干预处理（skip/rollback/approve）

     暂停拦截点:
     1. PLANNING.md 生成后 → 等待 confirm
     2. 质量校验失败后 → 等待人工干预
     3. 危险操作执行前 → 等待 approve/reject

     关键技术参考

     deepagents 框架使用模式（来自 HelloDeepAgents01.py）

     from deepagents import create_deep_agent

     # 1. 初始化 LLM
     llm = ChatOpenAI(api_key=..., base_url=..., model=..., temperature=0.2)

     # 2. 定义工具
     @tool
     def internet_search(query: str, ...):
         ...

     # 3. 创建 Agent
     agent = create_deep_agent(
         model=llm,
         tools=[internet_search],
         system_prompt="You are an expert researcher..."
     )

     # 4. 调用执行
     result = agent.invoke({"messages": [{"role": "user", "content": "..."}]})

  快速使用流程

  # 1. 初始化项目
  python -m deepagents init --name my-project

  # 2. 配置 workflow.md 和 agent.md

  # 3. 生成规划
  python -m deepagents run --phase plan

  # 4. 确认并执行
  python -m deepagents confirm --file PLANNING.md
  python -m deepagents run --phase execute

  # 5. 查看状态
  python -m deepagents status --live
